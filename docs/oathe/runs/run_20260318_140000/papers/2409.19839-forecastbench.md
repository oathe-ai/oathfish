# ForecastBench: Can LLMs Forecast?

**arxiv**: 2409.19839
**Published**: ICLR 2025
**Perspective**: SUPERFORECASTERS_VS_LLM

## Abstract

Forecasts of future events are essential inputs into informed decision-making. ML systems have potential to deliver forecasts at scale, but there is no framework for evaluating ML systems on standardized forecasting questions. This paper introduces ForecastBench: a dynamic benchmark that evaluates accuracy on an automatically generated and regularly updated set of 1,000 forecasting questions. To avoid data leakage, ForecastBench is comprised solely of questions about future events with no known answer at time of submission. Expert forecasters outperform the top-performing LLM (p-value < 0.001).

## Key Findings

1. **Expert forecasters significantly outperform best LLMs** (p < 0.001)
2. **State-of-art LLMs** (Claude 3.5 Sonnet, GPT-4 Turbo) perform roughly as well as median of forecasters with NO forecasting experience
3. **Dynamic benchmark**: 1,000 questions, continuously updated, no data leakage possible
4. **The gap is real**: Not a benchmark artifact — these are genuine future events
5. **Public leaderboard** at forecastbench.org for ongoing evaluation

## Key Brier Scores (from related work)

- Best superforecasters: ~0.096
- o3 (best LLM as of 2025): 0.1352
- Human crowd: 0.149
- Inexperienced human median: ~0.17-0.20
- Individual frontier LLMs: ~0.15-0.20

## OathFish Relevance

- **Proves single LLMs cannot match superforecasters** — validates need for structured architecture beyond prompting
- **The gap is fundamental**: LLMs lack real-time information access, deep causal reasoning, and calibrated uncertainty
- **OathFish must close this gap through ARCHITECTURE** (deliberation + ensemble + calibration), not just model capability
- **Benchmark for validation**: OathFish should submit predictions to ForecastBench leaderboard
- **Key question**: Does multi-agent deliberation actually close the gap? Paper doesn't test this
- **Superforecaster methodology** (decompose, base rate, update incrementally) should be codified into archetype reasoning
