# Decomposing Crowd Wisdom: Domain-Specific Calibration Dynamics in Prediction Markets

**arxiv**: 2602.19520
**Perspective**: CALIBRATION_SCIENCE

## Abstract

Prediction markets are increasingly used as probability forecasting tools, yet their usefulness depends on calibration — whether a contract trading at 70 cents truly implies a 70% probability. Using 292 million trades across 327,000 binary contracts on Kalshi and Polymarket, this paper shows that calibration is a structured, multidimensional phenomenon. On Kalshi, calibration decomposes into four components (universal horizon effect, domain-specific biases, domain-by-horizon interactions, trade-size scale effect) that together explain 87.3% of calibration variance.

## The Decomposition Model

Calibration slope modeled as:

```
θ(d, τ, s) = μ(τ) + α_d + β_d(τ) + γ_d(s) + ε
```

Where:
- **μ(τ)**: Universal horizon function — shared tendency toward underconfidence at long horizons
- **α_d**: Domain intercept — persistent bias (positive for Politics, negative for Weather)
- **β_d(τ)**: Domain-by-horizon interaction — how each domain deviates from universal curve
- **γ_d(s)**: Scale effect — how trade size modulates calibration within each domain

## Key Findings

1. **87.3% of calibration variance explained** by 4 structured components
2. **Domain-by-horizon interaction is largest domain-specific component** (26.0%)
3. **Persistent underconfidence in political markets**: prices compressed toward 50%
4. **Domain-agnostic models miss the largest source of calibration variation**
5. **Scale effect platform-specific**: Large trades = amplified underconfidence on Kalshi (Δ=0.53) but NOT on Polymarket (Δ=0.11)
6. **Bayesian hierarchical model confirms** with 96.3% posterior predictive coverage
7. **Data scale**: 292 million trades across 327,000 contracts

## OathFish Relevance

- **Calibration is STRUCTURED, not random**: Must track per-domain, per-horizon
- **Domain-specific biases are predictable**: Politics = underconfident, Weather = near-perfect
- **OathFish CALIBRATE phase must be domain-aware**: Per-archetype Brier scores alone are insufficient; need (archetype × domain × prediction_horizon) matrix
- **CRITICAL CHALLENGE**: Paper uses 292M data points. OathFish will have ~300-600 predictions. Formal decomposition requires orders of magnitude more data
- **Practical approach**: Track directional biases (over/under-confident per archetype per domain) from run 1, apply corrections at n=3+, don't attempt formal variance decomposition until n>100
- **"Consumers who treat prediction market prices as face-value probabilities will systematically misinterpret them"** — same applies to raw archetype predictions
